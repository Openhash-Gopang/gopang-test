## 목적

이 저장소는 고팡(Gopang) 웹앱을 실제 브라우저에서 테스트하기 위한 시나리오와 결과를 기록합니다.  
100명의 가상 사용자가 일상에서 고팡을 활용하는 500개 시나리오를 순서대로 테스트하고,  
누구나 이 문서를 참고해 독립적으로 테스트를 재현할 수 있도록 작성되었습니다.

## 테스트 환경 요구사항

### 브라우저
- Chrome 120+ 또는 Safari 17+ (권장: Chrome)
- 모바일: iOS Safari 17+ / Android Chrome 120+
- 개발자 도구(F12) 사용 가능 환경

### 네트워크
- 인터넷 연결 필수
- Supabase 대시보드 접근 권한 (주피터님 계정)

### 사전 준비
1. Chrome DevTools → Application → Local Storage 초기화
2. Supabase 대시보드 별도 탭 열어두기
3. 테스트할 사용자의 UUID와 전화번호 준비 (아래 users/ 폴더 참조)

---

## 진행 방법

### 순서
```
1. scenarios/Uxx_이름.md 파일 열기
2. 사전 준비 섹션 확인 및 환경 세팅
3. S-01부터 S-05까지 순서대로 브라우저 테스트
4. 각 시나리오 합격 기준 확인 후 결과 기록
5. 5건 완료 → results/summary.md 업데이트
6. 다음 사용자로 이동
```

### 결과 기록 방법
각 시나리오 파일 내 결과 란을 직접 수정합니다.
```markdown
- 테스트 결과: ✅ 통과  (날짜: 2026-06-13)
- 테스트 결과: ❌ 실패  (증상: 버튼 클릭 시 404 반환)
```

실패 시 `results/issues.md`에 버그를 추가합니다.

## 용어 설명

| 용어 | 설명 |
|------|------|
| GUID | 전화번호 기반 결정성 UUID (uuidv5) — 동일 번호 = 동일 GUID |
| JWT | gopang_token — 24시간 유효, localStorage 저장 |
| GDC | 고팡 디지털 화폐 (₮) — 결제 수단 |
| PDV | 거래 증거 데이터 — Hash Chain으로 무결성 보장 |
| fs_ledger | 금융 원장 — 모든 거래의 3행 기록 |
| ai_active | AI 비서 활성 여부 — user_llm_keys 기준 |
| handle | @읍면동_이름 형식의 업체 고유 URL 식별자 |
| entity_type | consumer(소비자) / org(사업자) / institution(기관) |
