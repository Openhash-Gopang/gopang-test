## S-03 — QR 결제 (pay.html)

### 상황
AI 비서 상담 후 짜장면과 짬뽕을 주문하기로 결정.  
사장 김민준이 생성한 결제 QR을 스캔하여 GDC로 결제합니다.

### 테스트 URL
```
https://users.gopang.net/pay.html?to=@hallim_geumneung&amount=15000&items=짜장면×1,짬뽕×1&expires=300&created_at=TIMESTAMP
```
> `TIMESTAMP` 자리에 현재 Unix timestamp 입력  
> 예: `https://currentmillis.com` 에서 확인 후 1000으로 나눈 정수값

### 브라우저 테스트 절차

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 위 URL 열기 | pay.html 결제 화면 표시 |
| 2 | 화면 내용 확인 | 업체명·금액·품목 중국어 표시 |
| 3 | 유효 시간 카운트다운 확인 | 05:00부터 감소 |
| 4 | GDC 잔액 확인 | 잔액 표시 (부족 시 S-03 건너뜀) |
| 5 | [서명하여 결제] 버튼 탭 | 서명 처리 중 표시 |
| 6 | 결제 완료 화면 확인 | 영수증 화면 (중국어) |

### 검증 데이터 (사전 계산 완료)
```
주문금액:        ₮15,000
플랫폼 수수료:   ₮450  (3%)
판매자 수령액:   ₮14,550
BIVM Σδ=0:      15,000 = 14,550 + 450  ✅
```

### Supabase 확인
```
Table: biz_orders
Filter: buyer_guid = '34c74aef-a50e-5c1d-be21-eae8e6d72225'
기대값: amount=15000, status='completed'

Table: fs_ledger
Filter: tx_id = {위 biz_orders의 tx_id}
기대값: 3개 행 (buyer debit / seller credit / platform credit)
```

### 합격 기준
- [ ] 결제 화면이 중국어로 표시
- [ ] 유효 시간 카운트다운 동작
- [ ] 결제 완료 후 영수증 화면 표시
- [ ] Supabase biz_orders에 레코드 생성
- [ ] Supabase fs_ledger에 3행 생성 확인
